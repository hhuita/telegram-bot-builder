# Analytics Dashboard — Roadmap

## Идея

Полноценная аналитика для Telegram-ботов прямо в редакторе.
Не просто 7 карточек — а живой дашборд с данными о пользователях, поведении, воронках и трафике.

---

## Что собираем

### Уровень 1 — Профиль пользователя (уже частично есть)

| Поле | Статус | Источник |
|------|--------|----------|
| `username`, `first_name`, `last_name` | ✅ есть | `from_user` |
| `is_premium` | ⚠️ баг — не пишется в БД | `from_user` |
| `is_bot` | ⚠️ не пишется в БД | `from_user` |
| `language_code` | ⚠️ не пишется в БД | `from_user` |
| `utm_source` | ⚠️ частично | `?start=` параметр |
| `deep_link_param` | ❌ нет | `?start=` параметр |
| `referrer_id` | ⚠️ в памяти, не в БД | `?start=ref_XXX` |
| `first_seen`, `last_seen` | ✅ есть | timestamp |
| `interaction_count` | ✅ есть | счётчик |

### Уровень 2 — Сообщения (из `bot_messages`, уже пишется)

- Всего сообщений от пользователя / от бота
- Типы сообщений: текст, фото, видео, документ, голос, стикер
- Какие `node_id` посещал пользователь
- Первое и последнее сообщение

### Уровень 3 — События поведения (нужна новая таблица `bot_events`)

- Какие кнопки нажимал и сколько раз
- Какие команды использовал (`/start`, `/help`, кастомные)
- На каком узле застрял / последний активный узел
- Количество сессий
- Среднее время сессии
- Время суток активности (утро / день / вечер / ночь)

### Уровень 4 — Воронки (агрегация по проекту)

- Сколько дошли от узла X до узла Y
- Где чаще всего выходят из бота
- Конверсия: зашли → ответили → дошли до конца
- A/B: какая ветка работает лучше

### Уровень 5 — Когорты (по времени)

- Retention: вернулись ли через 1 / 7 / 30 дней
- Новые vs вернувшиеся по дням
- Прирост базы по источникам трафика

## Новые таблицы

### `bot_events` — события поведения
```sql
CREATE TABLE bot_events (
  id            serial PRIMARY KEY,
  project_id    integer NOT NULL,
  token_id      integer NOT NULL,
  user_id       bigint NOT NULL,
  event_type    text NOT NULL,   -- 'button_click' | 'node_visit' | 'command' | 'session_start' | 'session_end'
  event_data    jsonb,           -- { node_id, button_text, command, ... }
  created_at    timestamp DEFAULT now()
);
```

### `bot_sessions` — сессии пользователей
```sql
CREATE TABLE bot_sessions (
  id            serial PRIMARY KEY,
  project_id    integer NOT NULL,
  token_id      integer NOT NULL,
  user_id       bigint NOT NULL,
  started_at    timestamp DEFAULT now(),
  ended_at      timestamp,
  duration_sec  integer,
  messages_count integer DEFAULT 0
);
```

### Расширение `bot_users`
```sql
ALTER TABLE bot_users ADD COLUMN is_premium     integer DEFAULT 0;
ALTER TABLE bot_users ADD COLUMN is_bot         integer DEFAULT 0;
ALTER TABLE bot_users ADD COLUMN language_code  text;
ALTER TABLE bot_users ADD COLUMN utm_source     text;
ALTER TABLE bot_users ADD COLUMN deep_link_param text;
ALTER TABLE bot_users ADD COLUMN referrer_id    text;
```

## Приоритизация

### Сделать прямо сейчас (баги и простые фиксы):
1. ✅ Записывать `is_premium`, `is_bot`, `language_code` в `bot_users`
2. ✅ Записывать `utm_source`, `deep_link_param`, `referrer_id` в `bot_users`
3. ✅ Компактные секции вместо 7 карточек в "Пользователях"

### Средний приоритет:
4. Таблица `bot_events` — сбор событий (node_visit, button_click, command)
5. Вкладка "Аналитика" — прирост, источники трафика, языки, топ узлы
6. Фильтр по источнику трафика в таблице пользователей

### Долгосрочно:
7. Таблица `bot_sessions` — сессии и retention
8. Воронки — конверсия по узлам
9. Heatmap активности по времени суток
10. A/B тесты веток
11. Экспорт в CSV / Google Sheets

## Технические детали

### Бэкенд
- Новый эндпоинт `GET /api/analytics/:projectId`
- Параметры: `?period=7d|30d|90d|all&tokenId=123`
- Кэширование 5 минут (Redis или in-memory)
- Библиотека графиков: `recharts` (уже в проекте)

### Фронтенд — новые компоненты
```
client/components/editor/analytics/
  AnalyticsPage.tsx           ← главная страница вкладки
  sections/
    GrowthChart.tsx           ← прирост базы (line chart)
    TrafficSourcesBar.tsx     ← источники трафика (horizontal bar)
    LanguagesBar.tsx          ← языки пользователей
    TopNodesBar.tsx           ← популярные узлы

    TopReferrers.tsx          ← топ рефереры
```

## Ограничения

- ⚠️ Данные только для проектов с включённой PostgreSQL
- ⚠️ Исторические данные — только с момента включения БД
- ⚠️ `bot_events` нужно добавить в шаблон генерации кода
- ⚠️ Retention требует `bot_sessions` — отдельная задача
